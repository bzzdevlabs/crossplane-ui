# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Big picture

`crossplane-ui` is a **monorepo** housing an admin UI for Crossplane v2, split
across four deliverables that ship together via one Helm chart:

```
┌─────────── services ───────────┐   ┌────── web ──────┐   ┌─── deploy ───┐
│  gateway (Go)  │  auth (Go)    │   │ ui (Vue 3)      │   │ helm chart   │
│  :8080 HTTP    │  :8081 HTTP   │   │ built, embedded │   │ + Dex dep    │
│                │               │   │ into gateway    │   │              │
└────────────────┴───────────────┘   └─────────────────┘   └──────────────┘
```

Two architectural invariants — both documented in ADRs — dictate most choices;
read them before proposing large changes:

- **Authorization is Kubernetes RBAC** via user impersonation
  ([ADR-0003](docs/adr/0003-k8s-impersonation-rbac.md)). The `gateway` holds a
  ServiceAccount with the `impersonate` verb and forwards the authenticated
  user's `sub`/`groups` via `Impersonate-User`/`Impersonate-Group` on every
  outbound call. There is **no second RBAC system** in application code.
- **Dex is the only auth surface** ([ADR-0004](docs/adr/0004-dex-as-auth-surface.md)).
  LDAP, SAML, OIDC, OAuth and local users all plug into Dex as connectors; our
  Go code only implements a single OIDC client.

Other decisions worth knowing: Crossplane v2 only ([ADR-0005](docs/adr/0005-crossplane-v2-only.md)),
Go + Vue 3 monorepo ([ADR-0002](docs/adr/0002-go-and-vue3-monorepo.md)).

## Request flow (gateway)

```
browser → Recover → RequestID → AccessLog → Metrics → CORS → mux
                                                                 ├── /healthz /readyz /metrics (no auth)
                                                                 └── /api/v1/* → AuthMW → api handlers
                                                                                      │
                                                                                      └─ kube.ClientFactory.For(user, groups)
                                                                                              │
                                                                                              ▼
                                                                                         K8s API (impersonated)
```

`AuthMW` is either `oidc.Middleware` (production) or `oidc.DevPassthrough`
(when `OIDC_ISSUER_URL` is empty — injects a synthetic `dev-admin` user).

`kube.ClientFactory.For` clones `base *rest.Config` per call with
`rest.ImpersonationConfig`, so concurrent requests never share headers.

## Common commands

Everything goes through `Taskfile.yml` — run `task` with no args to list
targets. Key ones:

| Task                  | What                                                                       |
| --------------------- | -------------------------------------------------------------------------- |
| `task bootstrap`      | Install Go tools + pnpm deps + pre-commit hooks (run once).                |
| `task lint`           | `golangci-lint` on every Go module + ESLint + Stylelint + helm + yamllint. |
| `task test`           | `go test -race -count=1` on every module + Vitest.                         |
| `task dev:up`         | Create k3d cluster (idempotent), install Crossplane, start compose stack.  |
| `task dev:down`       | Stop compose services (keeps cluster).                                     |
| `task dev:reset`      | Destroy compose + cluster + `compose/.kubeconfig`.                         |
| `task dev:logs`       | Follow compose logs.                                                       |
| `task dev:kubeconfig:export` | Print `export KUBECONFIG=…` for the host.                           |

The monorepo uses **Go workspaces** (`go.work`); run Go commands inside each
module's directory, e.g. `cd services/gateway && go test ./...` — at the repo
root `go build ./...` does not resolve because no module lives there.

Single-test examples:

```bash
cd services/gateway && go test -run TestNamespacesHandlerReturnsProjectedList ./internal/api/
cd web/ui && pnpm vitest run src/__tests__/auth.store.test.ts
```

## Local dev (k3d + compose)

The full stack lives under [`compose/`](compose/README.md). It joins the
`k3d-crossplane-ui` Docker network so services reach the API server at
`https://k3d-crossplane-ui-server-0:6443`. The kubeconfig mounted into
containers is generated by `task dev:kubeconfig` from `k3d kubeconfig get` with
the server URL rewritten to the internal DNS name and
`insecure-skip-tls-verify: true` (k3d's cert does not list the DNS name as a SAN).

Auth is in **dev pass-through mode by default** (no `OIDC_ISSUER_URL`) so
`curl http://localhost:18080/api/v1/namespaces` works without a token. Set
`OIDC_ISSUER_URL=http://localhost:5556/dex` in `compose/.env` to enable real
OIDC verification.

Port `8080` on the host is commonly taken; the compose gateway is remapped to
`18080` via `compose/.env`.

## Tooling pins and gotchas

- **Go 1.26**. Every `go.mod` and `go.work` carries a `toolchain go1.26.0`
  directive. If the installed `go` is older, Go auto-downloads 1.26 (requires
  `GOTOOLCHAIN=auto`, the default).
- **golangci-lint v2.x config** (`.golangci.yml` uses `version: "2"`). The
  upstream Docker image is still built with Go 1.25 (`golangci-lint#6272`);
  CI therefore rebuilds `golangci-lint` from source inside the Go 1.26 image.
  Local dev installs it via `go install` in `task bootstrap`.
- **gopls** versions ≥ 0.21 care about the `go 1.26` directive. If you see
  `go.work requires go >= 1.26 (running go 1.25.x)` in the IDE, **rebuild
  gopls with the current Go**: `go install golang.org/x/tools/gopls@latest`,
  then reload VS Code.
- **pnpm** is provisioned manually (Node 24+ dropped corepack bundling). The
  project has `"packageManager": "pnpm@10.x.x"` in `web/ui/package.json`; the
  dev installs pnpm via `get.pnpm.io/install.sh`.
- **`@vue/tsconfig`** v0.7 does **not** ship a `tsconfig.node.json`. Our
  `web/ui/tsconfig.node.json` extends `@vue/tsconfig/tsconfig.json` and
  redirects declaration output to `node_modules/.tmp/` to avoid stray
  `*.d.ts` at the repo root.
- **CGO** is required for `go test -race` (the race runtime links via cgo).
  GitHub's `ubuntu-latest` runner ships `gcc`; the workflow sets
  `CGO_ENABLED=1` before `go test`.

## CI scope (deliberately narrow right now)

`.github/workflows/ci.yml` currently ships only `lint` and `test` jobs. Jobs
for `build`, `package`, `scan` (Docker Scout; Trivy was dropped for the
known-vuln reason) and `release` (semantic-release on GitHub) are **present
but fully commented out** — uncomment them when the project is ready to
publish images to GHCR and cut releases. License-scanning was removed.

## Commits and conventions

- **Conventional Commits 1.0.0** enforced by `commitlint`. Scopes allowed:
  `gateway`, `auth`, `ui`, `chart`, `pkg`, `docs`, `adr`, `ci`, `deps`,
  `repo` (see `commitlint.config.mjs`).
- Go: `gofumpt` + `goimports` + `gci` via `golangci-lint run --fix`. Local
  prefix for import grouping:
  `github.com/bzzdevlabs/crossplane-ui`.
- UI: ESLint 9 flat config (`eslint.config.mjs`) + Prettier + Stylelint 17.
  Run `pnpm run lint:fix` to auto-format.
- Logs use `log/slog` JSON with `snake_case` keys (enforced by `sloglint`).

## When adding Crossplane-aware endpoints (M6+)

The gateway routes go through the `ClientFactory` and therefore honour
impersonation for free. For the home dashboard's aggregated sync/ready status,
watch for the Crossplane **v2** semantics (Operations, activation, namespaced
resources) — do not special-case v1 shapes, we only target v2 (ADR-0005).
